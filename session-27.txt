Role is a proper directory structure to write ansible playbooks. we can reuse roles. 
it will have tasks, vars, templates,files, handlers, etc...

Instead of writing one long playbook, you break it into smaller, reusable parts called roles. 
Each role handles one responsibility (like installing a web server, configuring a database, etc.).


What is inside a Role?

A role has a standard directory structure. The most common folders are:

my_role/
├── tasks/        # What to do (main logic)
├── handlers/     # Actions triggered by tasks (e.g., restart service)
├── templates/    # Dynamic config files (Jinja2 templates)
├── files/        # Static files
├── vars/         # Variables (high priority)
├── defaults/     # Default variables (low priority)
├── meta/         # Role metadata (dependencies)

####/roles/common/tasks/app-setup.yml


- name: create app directory
  ansible.builtin.file:
    path: /app
    state: directory
	
- name: create roboshop system user
  ansible.builtin.user:
    name: roboshop
    shell: /sbin/nologin
    system: true
    home: /app
	
- name: "download {{ component }} code"
  ansible.builtin.get_url:
    url: "https://roboshop-artifacts.s3.amazonaws.com/{{ component }}-v3.zip"
    dest: /tmp/{{ component }}.zip


- name: "extract {{ component }} code"
  ansible.builtin.unarchive:
    src: "/tmp/{{ component }}.zip"
    dest: /app

ansible templates
================= === ==================================

file --> content inside a specific file

template --> we can keep some placeholders, you can put the values at run time

follows jinja2 formatting, we can keep some placeholders, 
actual values will be provided through variables at runtime.

Ansible templates are dynamic files that use Jinja2 variables 
to generate customized configuration files automatically.

Role execution flow

When playbook runs:

Reads playbook
Finds role
Executes tasks
Loads variables
Applies templates
Runs handlers

  ansible.builtin.dnf:
    name: nginx
    state: present

    name: nginx
    state: started
    enabled: yes

  ansible.builtin.file:
    path: /usr/share/nginx/html
    state: absent

- name: create html directory
  ansible.builtin.file:
    path: /usr/share/nginx/html
    state: directory

- name: download frontend code
  ansible.builtin.get_url:
    url: https://roboshop-artifacts.s3.amazonaws.com/frontend-v3.zip
    dest: /tmp/frontend.zip

- name: unzip frontend code
  ansible.builtin.unarchive:
    src: /tmp/frontend.zip
    dest: /usr/share/nginx/html
    remote_src: yes

- name: remove default nginx conf
  ansible.builtin.file:
    path: /etc/nginx/nginx.conf
    state: absent

- name: copy roboshop nginx conf
  ansible.builtin.template:
    src: nginx.conf.j2
    dest: /etc/nginx/nginx.conf
  notify:
    - Restart nginx

REDIS:
======== ====================== ==================

roles/redis/tasks/main.yml

copy the redis.yml code

No code changes in redis.

- name: disable redis default module
  ansible.builtin.command: dnf module disable redis -y

- name: enable redis:7
  ansible.builtin.command: dnf module enable redis:7 -y

- name: install redis
  ansible.builtin.dnf:
    name: redis
    state: present

- name: allow remote connections
  ansible.builtin.replace:
    path: /etc/redis/redis.conf
    regexp: '127.0.0.1'
    replace: '0.0.0.0'

- name: disable protected mode
  ansible.builtin.lineinfile:
    path: /etc/redis/redis.conf
    regexp: 'protected-mode'
    line: protected-mode no
    
- name: enable and start redis
  ansible.builtin.service:
    name: redis
    state: started
    enabled: yes


## ansible-playbook -i inventory.ini -e "component=redis" main.yml
