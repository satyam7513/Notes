In ansible any modules not availible:
1. Devlop your own module using python
2. use shell or command module.


shell vs command

you are in your house
----------------------
1. I will enter into your home and call you
	problems: security issue, you are seeing everything inside house
	shell module --> if you are using shell module, you are logging inside the server and running the tasks.
	redirections, variables, pipes, etc will work here
	
2. I will stay outside and call you
	problem: you don't get any access to things inside
	command module --> executing command from outside. you will get linux full environment.
	redirections, piping and env variables will not work here.

#ansible.builtin.command: "ls -ltr > /tmp/output.txt"

add the shell line:

ansible.builtin.shell: "ls -ltr > /tmp/output.txt"

ansible-playbook -i inventory.ini 21-shell-command.yml

we can check the out-put, log into the node and check, but every time its very tough. 
so we are using register-keyword(stores the output)

👉 register is used to store the output of a task into a variable
so you can use it later in the playbook.

 
 register: command_result

  - name: print the outcome
    ansible.builtin.debug:
      msg: "{{ command_result }}"

ansible-playbook -i inventory.ini 21-shell-command.yml

  - name: redirect ls output to a file using command
    ansible.builtin.command: "ls -ltr"
    register: command_output
  
  - name: print the output
    ansible.builtin.debug:
      msg: "{{ command_output }}"
	  
ansible-playbook -i inventory.ini 21-shell-command.yml

Use command → whenever possible (safer, predictable)

Use shell → only when you need:
redirection (>)
pipes (|)
complex shell logic.

enabled=1
gpgcheck=0


mongodb.yml:
===== ========== ==================================== ===================================================

---
- name: configuring mongodb server
  become: yes
  hosts: mongodb
  tasks:
  - name: copy mongodb repo
    ansible.builtin.copy:
       src: mongo.repo
       dest: /etc/yum.repos.d/mongo.repo  # run the playbook and check its copied or not.

  - name: install mongodb server
    ansible.builtin.dnf:
        name: mongodb-org
        state: present

  - name: start and enable mongodb
    ansible.builtin.service:
       name: mongod
       state: started
       enabled: yes   #  run the playbook and check the port no: netstat -lntp

  - name: allow remote connections
    ansible.builtin.replace:
      path: /etc/mongod.conf
      regexp: '127.0.0.1'
      replace: '0.0.0.0'

  - name: restart mongodb
    ansible.builtin.service:
       name: mongod
       state: restarted
       
## ansible-playbook -i inventory.ini mongodb.yml
