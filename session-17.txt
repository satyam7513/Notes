#!/bin/bash

USERID=$(id -u)
R="\e[31m"
G="\e[32m"
Y="\e[33m"
N="\e[0m"
LOGS_FOLDER="/var/log/roboshop-logs" ### change the logs folder /roboshop-logs
SCRIPT_NAME=$(echo $0 | cut -d "." -f1)
LOG_FILE="$LOGS_FOLDER/$SCRIPT_NAME.log"

mkdir -p $LOGS_FOLDER
echo "Script started executing at: $(date)" | tee -a $LOG_FILE

# check the user has root priveleges or not
if [ $USERID -ne 0 ]
then
    echo -e "$R ERROR:: Please run this script with root access $N" | tee -a $LOG_FILE
    exit 1 #give other than 0 upto 127
else
    echo "You are running with root access" | tee -a $LOG_FILE
fi


# validate functions takes input as exit status, what command they tried to install
VALIDATE(){
    if [ $1 -eq 0 ]
    then
        echo -e "$2 is ... $G SUCCESS $N" | tee -a $LOG_FILE
    else
        echo -e "$2 is ... $R FAILURE $N" | tee -a $LOG_FILE
        exit 1
    fi
}

cp mongo.repo /etc/yum.repos.d/mongodb.repo
VALIDATE $? "Copying MongoDB repo"


cp mongo.repo /etc/yum.repos.d/mongodb.repo
VALIDATE $? "Copying MongoDB repo"

dnf install mongodb-org -y &>>$LOG_FILE
VALIDATE $? "Installing mongodb server"

systemctl enable mongod &>>$LOG_FILE
VALIDATE $? "Enabling MongoDB"

systemctl start mongod &>>$LOG_FILE
VALIDATE $? "Starting MongoDB"

sed editor --> stream editor
IN Mongodb we can change the bind portno 127.0.0.1 to 0.0.0.0 so 
we can use Sed editor .

sed (Stream Editor) is a Linux command-line tool used to read, edit, and transform text streams line by line.
It is commonly used to search, replace, delete, or modify text in files or command outputs.
Unlike editors like vi or nano, sed works non-interactively from the command line or inside scripts.


syntax: sed -i/-e 'expression' filename

example:
===== ===
goto ec2-user

cp /etc/passwd users
cat users

adding the line using sed:
sed -e '1 a hello world' users

check the output in cat users. hello world is not showing.

sed editor this is the main advanatage.

-e --> temp change onto the screen
-i --> perminent  change.

sed -i '1 a hello world' users --> after the 1st line.

sed -i '1 i hello world' users -- insert before the 1 st line.

sed -i '1 d' users --> line no 1 deleted.

sed -e 's/root/ADMIN/g' users --> root replace with admin.

sed -e '/overflow/ d' users --> overflow word deleted

sed -i 's/127.0.0.1/0.0.0.0/g' /etc/mongod.conf

Idempotency:
======= ====
Irrespective of the number of time you run something it should not change the result.

API Development.

post (not idempotency)--> first time entry willbe created ,second time you entry on the same credentials its shows error.

so first check user already exist or not,if not create otherwise show the error.

Get --> Idempotency(Every time you run it don't change the result)
Put(Update)---> idempotency, but for better experience, same details are already updated.
DELETE ---> Idemoptent, but for better user experience, user already deleted.

** second time you run this script through error.why because roboshop user already craeted,
so we can add check user already exist or not.
