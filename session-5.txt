package management
====================
What is Package Management?

👉 Package management is a system used in Linux to:

Install software

Update software

Remove software

Manage dependencies (required libraries/files)

Maintain software repositories

📌 Instead of downloading software manually, Linux uses package managers.

(package manager connects to internet windows website. downloads them and install them)

Types of Linux Package Managers:


Linux distributions use different package systems:

Distribution	             Package Manager	                        Package Format
Ubuntu / Debian	             APT (apt, apt-get)                     	  .deb
RHEL / CentOS / Fedora	      YUM / DNF	                                  .rpm


if you want install any package must use root access.

dnf install <package name>
dnf install git

sudo apt install package_name
sudo apt install nginx

sudo apt remove package_name --> Removes package but keeps config files.
sudo apt purge package_name  --> Removes package + configuration files.

apt search package_name -> search package
apt show package_name --> show package details.


dnf remove –y <package-name>  –> remove the package

/etc/yum.repos.d  -->  repos available in that location

dnf list installed -->  shows the installed packages list

dnf list available --> shows the available packages

dnf list avaialible | grep python  --> shows the python packages.




service management
===================
What is Service Management?

👉 Service management in Linux is used to:

Start a service, Stop a service, Restart a service, Enable service at boot, Disable service at boot, Check service status

📌 Services = background processes (daemons)

Examples: sshd, nginx, httpd, postgresql, docker.

Service Management Systems in Linux: 

systemd	    -->    RHEL 7+, CentOS 7+, Ubuntu 16+, Fedora

systemctl is used to manage services in systemd.

systemctl <action> <service_name>


systemctl start <service-name>
systemctl stop <service-name>
systemctl status <service-name>
systemctl restart <service-name>
systemctl enable <service-name>
systemctl disable <service-name>

install nginx and start it : dnf install nginx -y

Nginx port number-80

service files availible in :
 /usr/lib/systemd/system/
/etc/systemd/system/




1.Difference between start and enable:

Starts the service immediately

Service runs only for the current session

Enables the service to start automatically at boot

Does not start the service immediately

one-liner:
👉 start controls runtime, enable controls boot time

2.What is systemd?

systemd is that  manages services and system startup

3.Where are service files stored?

 /usr/lib/systemd/system/   → Default system services
/etc/systemd/system/       → Custom or overridden services

4.How do you troubleshoot a failed service?

  1️ Check service status

    systemctl status service_name
	
  2.Check service logs

     journalctl -u service_name
	 
  3.Check config files

     Syntax errors

      Wrong paths

     Port conflicts
	 
  4.check dependencies

       systemctl list-dependencies service_name
	   
  5.Restart service

       systemctl restart service_name
	   
one-liner:
 I check status, logs, configuration, and dependencies using systemctl and journalctl.
 
 
 5.Difference between reload and restart?
 
    systemctl reload nginx
	
  Reloads configuration files

  Service keeps running - No down time.
  
  restart:
  
  systemctl restart nginx
  
  Stops and starts the service again

  Causes brief downtime
