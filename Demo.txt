What is SDLC?

SDLC = Software Development Life Cycle

It is a step-by-step process used to build software in a planned and structured way.

SDLC phases :

Requirement – What does the customer want?

Design – How will we build it?

Development – Writing the code

Testing – Checking for bugs

Deployment – Releasing to users

Maintenance – Fixes & improvements

waterfall
===========

Waterfall Model

Waterfall = Step by step (one phase after another)

How it works:
Requirement → Design → Development → Testing → Deployment → Maintenance


You finish one phase completely before moving to the next.

Key points:

No going back easily

Changes are difficult

Everything is planned at the beginning

Best for:

Small projects

Clear & fixed requirements.

Pros:

✔ Easy to understand
✔ Clear documentation

Cons:

❌ Very slow
❌ Not flexible to changes



Government or legacy systems
once requirements are fixed we can't change

Agile
============================================================
3️⃣ Agile Model

Agile = Build in small parts, fast & flexible

Instead of building everything at once, we build small features in short cycles (sprints).

How it works:

Work is divided into sprints (2–3 weeks)

Each sprint includes:

Plan

Develop

Test

Deliver

Key ideas:

Customer feedback is continuous

Changes are welcome

Frequent releases+


Pros:

✔ Fast delivery
✔ Easy to handle changes
✔ Customer is involved

Cons:

❌ Needs strong team coordination
❌ Less documentation
